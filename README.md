# 法典 — AI 数字法务团队

> EasyClaw Link 黑客松参赛作品 · 2026

---

## 🎯 一句话介绍

**「不替你做决定，只帮你不漏掉什么。」**

法典是一支由 7 个 AI Agent 组成的数字法务团队。它会读合同、查法规、标风险、出报告——但永远把最终判断留给你和你的律师。

---

## 🛡️ 设计理念

### 不是 AI 法官，是 AI 法务助理

法学界对 AI 参与法律行业的主流担忧是「机械断案、无法实现个案正义」。本项目的定位避开了这个陷阱：

| ❌ AI 不能做的 | ✅ AI 能做的 |
|---------------|-------------|
| 判断合同能不能签 | 标注17个需要人工确认的风险点 |
| 预测官司能不能赢 | 整理相关法条和类案裁判要点 |
| 替代执业律师 | 帮律师省掉翻合同找条款的体力活 |

### 安全边界（内置，不可绕过）

- 不下法律结论——只标注风险、提供参考
- 每条产出强制附带免责声明
- 不编造法条——不确定的宁可不写
- 涉及刑事/人身安全/重大财产——强制提示咨询专业律师
- 不处理未经脱敏的真实个人信息
- 总指挥验收机制——子Agent产出必须通过总指挥核对

## 🏛️ 团队架构

| 角色 | Agent ID | 职责 |
|------|----------|------|
| ⚖️ 法典·总指挥 | `commander` | 接收指令、拆分任务、调度全队、质量验收 |
| 🔍 合同审查官 | `creator` | 读合同、逐条标注风险等级、输出审查清单 |
| 📚 法规研究员 | `researcher` | 检索法律法规、整理判例、提供法条依据 |
| 📋 合规顾问 | `compliance` | 整合审查+法规、出合规报告、协商优先级 |
| ✍️ 文书起草员 | `drafter` | 起草合同、律师函、法律意见书 |
| ⚖️ 案例分析官 | `analyst` | 类案检索、判例要点整理、案例对比 |
| 📎 证据梳理员 | `evidence` | 证据链梳理、时间线、材料清单 |

每个角色有独立的 SOUL.md（人设、职责、安全边界），总指挥有完整的 task-routing-rules.md。

---

## 🔄 协作流程（Pipeline）

```
合同进入 → 🔍审查官（标风险）
              ↓
           📚研究员（查法条）
              ↓
           📋合规顾问（出报告）
              ↓
           ⚖️总指挥验收 → 交付
```

三审合一，全自动。单个合同审查耗时约 3分30秒。

## 🚀 部署方式

### 前置要求
- EasyClaw 已安装并运行
- DeepSeek API Key 已配置

### 步骤
1. 将 `agents/` 下各目录复制到 `~/.easyclaw/workspace/agents/<role>/`
2. 在 EasyClaw 配置中注册 7 个 Agent（见 agent-config.json）
3. 通过 `sessions_spawn` 向 commander 派发法务任务
